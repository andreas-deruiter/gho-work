Documents Panel: What should it show?

The Documents panel lives in the sidebar. What role does it play for the user?

A

Full Explorer Clone

Show every file and folder in the workspace — no filtering. The user treats GHO Work like a file manager. Familiar to VS Code users, but may surface noise (dotfiles, build artifacts, node_modules) that isn't relevant to office work.

Pros

  • Familiar VS Code behavior
  • Nothing hidden, full transparency
  • Simplest to implement

Cons

  • Noisy for non-developer users
  • Dotfiles and build artifacts clutter the view
B

Smart Explorer (Recommended)

Show the full tree but with sensible defaults: auto-hide dotfiles and build artifacts (.git, node_modules, etc.), surface office-relevant files more prominently. Users can toggle "Show All Files" to see everything.

Pros

  • Clean default view for office users
  • Full power available via toggle
  • Best of both worlds

Cons

  • Slightly more complex (filter logic)
  • Users might not find hidden files initially
C

Document-Focused View

Only show files the agent created or the user explicitly added. A curated, minimal view — more like a "Recent Documents" panel than a file explorer.

Pros

  • Very clean, zero noise
  • Focused on agent output

Cons

  • Can't browse existing files
  • Limits discoverability
  • Different mental model from Explorer
