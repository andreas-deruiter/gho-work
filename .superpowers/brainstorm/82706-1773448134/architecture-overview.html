Documents Panel — Architecture Overview

How the pieces fit together across the monorepo

Component Architecture
RENDERER (packages/ui + apps/desktop)
DocumentsPanel extends Widget
  ├─ header (title + refresh + new file btn)
  ├─ filter input
  ├─ TreeWidget<FileNode>
  │   ├─ recursive expand/collapse
  │   ├─ file-type icons (by extension)
  │   ├─ attach button per row
  │   ├─ right-click → context menu
  │   └─ click → info popover
  └─ status footer (file count)
TreeWidget<T> (new, reusable)
  ├─ ITreeDataSource<T>
  ├─ ITreeRenderer<T>
  ├─ expand/collapse state
  ├─ keyboard nav (↑↓←→Enter)
  └─ onDidSelect / onContextMenu events
ChatInput (existing, modified)
  └─ attachment chips area
     ├─ file icon + name + remove btn
     └─ sent with message payload
MAIN PROCESS (packages/platform + electron)
IFileService (existing interface, new impl)
  ├─ readDir(path) → FileEntry[]
  ├─ stat(path) → FileStat
  ├─ createFile / createDir
  ├─ rename(old, new)
  ├─ delete(path)
  └─ watch(path) → file change events
NodeFileService implements IFileService
  └─ Node.js fs/promises + chokidar watcher
IPC Handlers (new channels)
  ├─ files:readDir
  ├─ files:stat
  ├─ files:create
  ├─ files:rename
  ├─ files:delete
  ├─ files:watch / files:unwatch
  └─ files:changed (main→renderer)
DATA FLOW
1. User clicks Documents icon → DocumentsPanel renders with workspace root
2. Panel calls window.ghoWorkIPC.invoke('files:readDir', path)
3. Main process NodeFileService reads directory, returns FileEntry[]
4. TreeWidget renders entries; expanding a folder triggers another files:readDir
5. File watcher sends files:changed events → tree auto-refreshes affected subtree
6. User clicks attach button → file reference added as chip in ChatInput
7. Message sent with attached file paths → agent reads file content via SDK tools

Key Design Decisions

Lightweight TreeWidget

~500-800 lines, purpose-built. Recursive DOM, no virtualization. Expand/collapse, keyboard nav, events. Reusable for other tree UIs later.

Lazy loading

Children loaded on expand, not upfront. Keeps initial render fast for large workspaces. Cached until file watcher invalidates.

Smart defaults

Hide dotfiles and build artifacts by default. "Show all files" toggle in header. Filter patterns configurable in settings later.

Attach, don't open

v1 has no document preview. Clicking attach adds a chip to ChatInput. Clicking the file name shows info (size, modified, reveal in Finder). Preview comes later.
