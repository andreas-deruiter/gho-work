How should Connectors live in the workbench?

The sidebar shows quick status; the settings panel has full configuration. How do they relate?

A

Sidebar + Settings in Main Area

Clicking "Connectors" in the activity bar shows a quick-view sidebar (status dots, toggles, tool counts). A gear icon in the sidebar header opens the full settings panel in the main content area (replacing the chat). This matches VS Code's pattern: sidebar for browsing, editor area for details.

Pros

  • Familiar VS Code pattern
  • Sidebar stays visible while configuring
  • Full width for forms and tab content

Cons

  • Navigating away from chat to configure
  • Two separate UI areas to maintain
B

Sidebar Only (Expandable)

Everything lives in the sidebar. The default view shows the connector list. Clicking a connector expands it inline to show tools, toggle, and a "Configure" section. Adding a new connector opens a form within the sidebar. No separate settings panel.

Pros

  • Single location for everything
  • Chat stays visible while managing connectors
  • Simpler navigation

Cons

  • Sidebar width (240px) is tight for forms
  • Deep nesting: list → detail → tools → config
  • Scrolling issues with many connectors
C

Sidebar + Slide-over Panel

Activity bar shows the connector sidebar (list view). Clicking a connector or "Add New" opens a slide-over panel from the right (like a drawer, 400px wide) that overlays the main content. Chat remains underneath. Close the drawer to go back.

Pros

  • Chat stays visible (partially)
  • More room than sidebar for forms
  • Clear "I'm configuring" mode

Cons

  • New UI pattern not in the app yet
  • Overlay can feel modal/blocking
  • More complex implementation
