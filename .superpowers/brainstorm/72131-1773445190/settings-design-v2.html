Settings Panel Design — Updated Architecture

Shell + separate page widgets. Visual design unchanged.

Component Structure

File Layout & Ownership
packages/ui/src/browser/settings/
settingsPanel.ts — Shell widget: nav list + content container
• Owns the nav ↔ page mapping
• Mounts/unmounts page widgets on nav click
• Disposes previous page, creates new one
appearancePage.ts — Theme card selector
• Receives IThemeService via constructor
• Renders theme cards, handles selection
• Self-contained, independently testable
skillsPage.ts — Skill browser + path config
• Receives IIPCRenderer via constructor
• Fetches skill list & paths via IPC
• Add/remove paths, rescan, grouped list
settingsPage.ts — Base interface for all pages
• interface ISettingsPage extends IDisposable { getDomNode(): HTMLElement }
• Future pages (Models, Permissions, etc.) implement same interface

Shell ↔ Page Interaction

How SettingsPanel manages pages
Nav click "Appearance":
1. If activePage exists → activePage.dispose()
2. Create new AppearancePage(themeService)
3. Append page.getDomNode() to content container
4. Store as activePage
Nav click "Skills":
1. activePage.dispose() → removes AppearancePage
2. Create new SkillsPage(ipc)
3. SkillsPage.constructor fetches data via IPC
4. Append to content container, store as activePage
Settings panel disposed (user clicks away):
activePage.dispose() + shell cleanup via DisposableStore
