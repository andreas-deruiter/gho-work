# Install CLI for Microsoft 365 (m365)

## What this tool does
Enables GHO Work to manage Microsoft 365 services: SharePoint sites, Teams channels, Planner tasks, Power Platform apps, and more.

## Important: You are the installer

The user clicked "Install" because they want YOU to handle this. Do not tell the user to run commands — run them yourself using your bash tool. The user should only need to sign in via their browser.

## CRITICAL: Install `m365`, not `mgc`

These are different tools:
- `m365` = CLI for Microsoft 365 (PnP community tool) — THIS is what you're installing
- `mgc` = Microsoft Graph CLI (Microsoft's official tool) — NOT this one

## Step 1: Check current state

Run `m365 version`. If it fails, install it. If it succeeds, skip to Step 3.

## Step 2: Install

Node.js 18+ is required. Check with `node --version`.

Run: `npm install -g @pnp/cli-microsoft365`

Verify: `m365 version`

## Step 3: Authenticate

**No app registration is required.** The m365 CLI ships with a default app ID (PnP Management Shell) that works out of the box. Do NOT ask the user to create an Entra app registration. Do NOT run `m365 setup`. Do NOT try alternative app IDs.

**CRITICAL: Use device code flow only.** Run exactly this command:

```
m365 login --authType deviceCode
```

This command will output a URL and a code. Parse the output for:
- A URL (https://microsoft.com/devicelogin)
- A code (letters and numbers, like `ABCD1234`)

**Show the code to the user FIRST, prominently and clearly.** Then tell them to open the URL in their browser, enter the code, and sign in with their Microsoft account.

Wait for the command to complete. It will exit when authentication succeeds.

**If the command exits immediately without printing a code:** the user may already be authenticated. Run `m365 status` to check. If it says "Logged in", authentication is already done.

## Step 4: Verify

Run `m365 status`. It should say "Logged in as ...".

Tell the user the result: installed, authenticated, ready to use.

## Common pitfalls
- **Do NOT ask the user to create an app registration** — the default PnP Management Shell app ID works fine
- **Do NOT run `m365 setup`** — it's unnecessary for basic authentication
- **Do NOT use browser-based auth** — always use `--authType deviceCode`
- Permission errors on npm install → use `sudo npm install -g` on macOS/Linux
- Tenant restrictions → admin may need to consent to the PnP Management Shell app in Entra ID
- Multiple accounts → `m365 logout` first, then re-authenticate
